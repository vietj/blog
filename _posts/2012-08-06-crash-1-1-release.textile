h3. Spring support


CRaSH can be embedded and configured as a Spring bean providing automatic access to your Spring beans. Writing CRaSH commands that leverage Spring beans is now trivial, now you can extend your Spring applications with CRaSH!

h3. Standalone mode improvements


The standalone mode has a few improvements:
<ul>
	<li>a conf/crash.properties has been added for configuring the shell</li>
	<li>the SSH connector has been added</li>
</ul>

h3. Download



CRaSH can now be fully downloaded and we also provide smaller archives depending on the bundle you like:
<ul>
	<li><a href="https://code.google.com/p/crsh/downloads/detail?name=crsh-all-1.1.0.tar.gz&can=2&q=">CRaSH All</a> : all the available bundles</li>
	<li><a href="https://code.google.com/p/crsh/downloads/detail?name=crsh-standalone-1.1.0.tar.gz&can=2&q=">CRaSH Standalone</a> : starts a JVM or attach to it a running JVM</li>
	<li><a href="https://code.google.com/p/crsh/downloads/detail?name=crsh-web-1.1.0.tar.gz&can=2&q=">CRaSH Web</a> : the classic web application bundle</li>
	<li><a href="https://code.google.com/p/crsh/downloads/detail?name=crsh-spring-1.1.0.tar.gz&can=2&q=">CRaSH Spring</a> : embedded as a Spring bean in a web application</li>
	<li><a href="https://code.google.com/p/crsh/downloads/detail?name=crsh-gatein-1.1.0.tar.gz&can=2&q=">CRaSH GateIn</a> : extends the web bundle and adds specific GateIn features like Java Content Repository support</li>
</ul>
